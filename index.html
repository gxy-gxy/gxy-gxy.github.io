苏州大学计算机科学与技术学院学生科技协会科研部活动策划案

Posted on 2090-02-16 Edited on 2020-02-16

Debug交流会

活动背景

大一下学期，计科院的学生将开始学习使用编程语言解决更实际的问题，而不再停留于语言本身。人工智能实验班有数据结构，需要实现各种各样的容器和算法；其他专业的学生也会开始学习静态语言。在这一过程中，一定会出现许多上学期考虑不到的编程问题（bug）。学生们需要在遇到bug
时有一个交流的平台。

活动目的

帮助计科院19 级新生在大一下学期解决一些自己暂时无法解决的bug，也可通过他人的bug 对自己的编程习惯进行指导。此外，对于软件工程专业的学生来说，还可以验证大一上学期学习到的一些设计模式的正确性。

活动名称

Debug 交流会

活动方案

活动时间

未知，两小时左右

活动地点

鸿远楼/博远楼，1∼2 间教室。

参与对象

科研部
19级新生

活动内容

  • bug 收集
  • bug 交流
  • bug 解决
  • bug 启示

活动流程

  • 宣传
    • 联系宣传部，做好微信和QQ 推送
    • 联系图文中心制作宣传海报
    • 联系辅导员和通知大一新生
  • 收集bug
    依托于Github 平台，收集并统计同学提交的bug，待bug 数量足够时，可以着手准备交流。
  • 准备
    确定好讲师、主持等人员，借好教室，检查好设备
  • 交流
    首先由bug 提出者一一讲述自己的问题，若已经解决还可以讲述解决的过程。然后由讲师解决bug。待所有bug 都解决完毕后，讲师总结bug 中出现的编程风格或设计模式方面的问题，并给出相应的建议。
  • 后期处理
    • 清理教室
    • 新闻稿和总结稿的撰写

人事安排

  • 联系宣传部
  • 联系图文中心
  • bug 收集与统计
  • 借教室
  • 主持人
  • 主讲人
  • 新闻稿
  • 总结稿
